Idea to deploy
01

Set up your project in 3 steps

Create the folder

Every project starts with a clean folder. This becomes your workspace, your repo, and your deployment source.

Add project intelligence

CLAUDE.md tells the AI who you are, what you are building, and how to behave. SKILL.md, AGENTS.md, and memory files give it depth.

Open the terminal

Run Claude Code in your project folder. The AI reads your files, understands your intent, and starts building.

02

Teach the AI your project

CLAUDE.md

The main briefing

Rules, structure, tone, what is locked, what is open. The single source of truth for every conversation.

SKILL.md

Deep context

Strategy, audience, evidence, assets. Everything the AI needs to make informed decisions about your project.

Memory System

Persistent files that help the AI remember across conversations. User preferences, feedback, project state.

Agents

Specialized sub-agents for different tasks. Visual design, build, motion, quality review. Each with its own tools and focus.

03

Build, preview, iterate, ship

1

Describe your vision

Tell the AI what you want. Be specific about the feeling, not just the features.

2

Watch it build

Open a local preview. The AI writes HTML, CSS, JS in real time. You see every change instantly.

3

Guide with feedback

Screenshot what you see. Say what to fix. The AI reads your files before every edit.

4

Commit and push

The AI commits to GitHub with clear messages. Push triggers automatic deployment to Vercel.

5

Live in minutes

Your site is live. Custom domain, SSL, CDN. From idea to production in one session.

04

Multiple agents, one vision

Complex projects use specialized agents working in parallel. A build agent writes code while a visual designer reviews composition. A motion engineer handles animations while a quality reviewer checks everything.

Parallel execution

Launch multiple agents at once for independent tasks. They work simultaneously without blocking each other.

Specialized roles

Each agent has its own tools and expertise. Build, design, motion, quality. Focus creates better output.

Shared context

All agents read the same project files and follow the same rules. Consistency without coordination overhead.
