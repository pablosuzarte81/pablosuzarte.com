Designing Project Intelligence

A future role for people who design how humans and AI think together.

The focus of this master-program-style website is not only to explain a framework. It is to help people build a new mindset from 0 to 100 and design their own project intelligence.

At the center of that role is a simple idea: a project brain is not a folder of notes. It is an operating system for human-AI collaboration.

Role definition

Single source of truth logic designer

Someone who turns repeated friction into the right file, structure, and workflow layer so serious projects can continue with clarity and low cognitive friction.

Operating rule

Structure should emerge from friction

If friction appears repeatedly, create the file or structure that solves that friction. Real projects teach what structure is necessary.

01

Overview

AI Project Brain Lab is where I explore the overlap between AI, UX, systems thinking, and education.

The project started from a simple observation: most AI workflows look impressive in the moment, but collapse when a project becomes complex, collaborative, and long-running.

Instead of teaching prompting as a trick, I shaped this as a course concept about context design, continuity, and project structure. The result is both a teaching product and a portfolio artifact for how I think about future-facing design work.

02

Problem

Context disappears

People keep rebuilding the same background instead of building momentum across sessions.

Continuity breaks

Good thinking gets lost between notes, tools, and decisions, which makes serious work hard to continue.

Outputs stay shallow

AI can sound useful in moments, but it often fails when a project needs memory, judgment, and structure over time.

03

Shift

Old mode

Reactive AI use

  • Ask isolated questions
  • Hope the output fits
  • Repeat project context constantly
  • Lose clarity between sessions

New mode

Designed AI collaboration

  • Design context deliberately
  • Build continuity into the workflow
  • Separate evidence, status, and decisions clearly
  • Guide intelligence across time
04

System

The core teaching device is a five-layer project brain that makes serious AI collaboration more usable.

CLAUDE.md

Defines the project worldview, intent, and reasoning path.

PROJECT_STATUS.md

Makes progress, current state, and next steps legible.

EVIDENCE_PACK.md

Separates grounded proof from loose opinion.

ASSET_INVENTORY.md

Tracks what exists so the work can build on real material.

WORKING_STYLE.md

Clarifies tone, collaboration preferences, and execution standards.
