Claude Code 101

From prompting a chatbot to orchestrating agents that solve real problems.

Duration 3–4 hours
Level Beginner → Advanced

The mindset before the tool

Most people prompt a chatbot. By the end of this workshop, you will orchestrate agents to handle complex, multi-step tasks. The difference is not the tool — it is the environment you design around it.

This workshop is built on Project Intelligence Principles — the same framework I use to set up human-AI collaboration at scale. Each module maps to a principle.

01Shared Memory→CLAUDE.md & project setup
02Current Orientation→First conversation & context
03Evidence Discipline→Reading & understanding code
04Collaboration Clarity→Permissions & working agreements
05Role Clarity→Custom agents & slash commands
06Decision Continuity→Memory system & persistence
07Asset Awareness→MCP servers & external tools

Setup

Before the workshop

Participants need these ready before we start:

That's it. First run opens a browser to log in. After that, you're in.

01

Shared Memory — teaching AI who you are

Principle: AI needs a clear understanding of what the project is, why it matters, and what must not be misunderstood.

What you will do

Create a CLAUDE.md file — the shared memory between you and AI. This is not a README. It is a living agreement about what is true.

Why it matters

Without shared memory, every conversation starts from zero. With it, Claude Code understands your project, your stack, your conventions, and your constraints from the first message.

Try it — 15 min

Inside Claude Code, run this — it scans your project and generates a CLAUDE.md for you:

/init

Then edit the generated file together. Make sure it answers:

  1. What is this project? (one paragraph)
  2. What is the tech stack?
  3. What commands matter? (dev, build, test, deploy)
  4. What conventions should never be broken?
  5. What should AI never do? (delete files, push to main, etc.)

Example CLAUDE.md — copy and adapt for your project

# My Project

## Stack
- Next.js 15, TypeScript, Tailwind CSS
- Deployed on Vercel

## Commands
- npm run dev — start dev server
- npm run build — production build

## Conventions
- All components in src/components/
- Never modify package-lock.json manually
- Do not push to main without review
Key insight: CLAUDE.md is automatically loaded into every conversation. It is the most important file in your project for AI collaboration.

02

Current Orientation — your first real conversation

Principle: The project needs a visible sense of where things stand now.

What you will do

Have your first meaningful conversation with Claude Code. Not "write me a function" — but "understand where I am and help me move forward."

Try it — 15 min

Open your project in terminal. Copy and paste these one at a time:

claude

Once inside, try these prompts progressively:

Read the project and tell me what you understand about it
What are the main files and how do they connect?
I want to add a dark mode toggle. What is your approach before writing any code?

Watch what happens. Claude reads your files, understands your structure, and reasons about your project. This is not a chatbot — it is an agent with tools.

ReadReads any file in your project
GlobFinds files by pattern (*.tsx, src/**/*)
GrepSearches content across files
BashRuns terminal commands
EditMakes precise changes to files
WriteCreates new files
Key insight: Claude Code is not generating text — it is using tools to explore, understand, and modify your actual codebase. The quality of its work depends on how well you orient it.

03

Evidence Discipline — read before you write

Principle: Ideas should be grounded in visible evidence, not assumptions.

What you will do

Learn to ask Claude Code to investigate before acting. The most common mistake beginners make: asking AI to write code without understanding the existing code first.

Try it — 20 min

Pick a real file in your project. Before asking Claude to change anything, make it investigate first:

Read src/components/Header.tsx and explain how it works
How do other components in this project handle navigation?
Now implement the change, following the patterns you found

Replace the file paths with your own. The pattern is always: read → understand → then act.

Key insight: An AI collaborator will treat an assumption as a fact if you present it as one. The discipline to say "read first, then act" produces dramatically better results.

04

Collaboration Clarity — permissions and trust

Principle: Good results depend on knowing how to work together well.

What you will do

Configure how Claude Code works with you. Permissions are not restrictions — they are working agreements.

Try it — 10 min

Inside Claude Code, check your current permissions:

/permissions

Now try something that needs approval. When prompted, notice you can allow it just once, for the session, or permanently.

Pro tip: press Shift+Tab to cycle between permission modes (Default → Auto-accept edits → Plan mode).

Key insight: Start strict, loosen as trust builds. This mirrors how you onboard any new team member — you give more autonomy as they prove they understand the context.

05

Role Clarity — custom agents and commands

Principle: Every collaborator should have a clear role and responsibility.

What you will do

Create custom slash commands that give Claude Code specific roles. Instead of one general AI, you design specialised agents.

Try it — 20 min

Create your first custom command. Run these in your terminal (not inside Claude):

mkdir -p .claude/commands

Now paste the review.md content from below into .claude/commands/review.md. Then go back into Claude Code and type:

/review

That's it. You just created a specialist agent.

Example: .claude/commands/review.md — copy this to get started

You are a senior code reviewer. Review the staged
changes (git diff --cached) and provide feedback on:

1. Code quality and readability
2. Potential bugs or edge cases
3. Performance considerations
4. Whether the changes follow the project conventions
   defined in CLAUDE.md

Be direct. If it looks good, say so briefly.
Key insight: Custom commands turn Claude Code from a general assistant into a team of specialists. A /review agent, a /test agent, a /deploy agent — each with a clear role.

06

Decision Continuity — memory that persists

Principle: Decisions and reasoning should be preserved so the team does not restart the same thinking.

What you will do

Learn how Claude Code's memory system works. Conversations end, but important decisions should survive.

Try it — 15 min

Inside Claude Code, make a decision and ask it to remember:

Let's use server components for all data fetching. Remember this decision.

Now clear the conversation and test if it stuck:

/clear
What are our design decisions?
Key insight: The biggest failure mode in AI-assisted projects is losing context between sessions. CLAUDE.md + the memory system solve this — decisions carry forward, not just code.

07

Asset Awareness — connecting external tools

Principle: Both humans and AI need to know what materials exist.

What you will do

Connect Claude Code to external services using MCP (Model Context Protocol) servers. This extends what Claude can see and do.

Try it — 15 min

Inside Claude Code, open the settings:

/settings

Add an MCP server — Vercel, GitHub, or a file system server. Then try something like:

Check my latest Vercel deployments

Claude now has awareness of systems outside your codebase. That is asset awareness.

Key insight: MCP servers turn Claude Code from a code assistant into a workspace assistant. It can read your deployments, your project management tools, your documentation — anything you connect.

08

Reusable Patterns — orchestrating complex tasks

Principle: Workflows that work well should be saved, refined, and reused.

What you will do

This is where it all comes together. You will orchestrate a complex, multi-step task — the kind that would take hours manually but minutes with a well-designed AI environment.

Try it — 30 min

This is the big one. Pick an ambitious task and give Claude the full context:

Add a contact form to this site with validation and an API endpoint that sends an email. Read the project first, plan your approach, then implement it.

Watch what happens. Claude will read your project, propose an approach, create files, edit existing ones, and handle errors — all autonomously. When it's done:

/commit

Claude writes the commit message from the actual changes. Review the git diff together.
